# ASUS 伺服器市場拓展：難易度評分標準（Difficulty Scoring Rubric）

## 評分架構

難易度評分為 **1-10 分**（1 = 最容易，10 = 最困難），由以下 **6 個維度** 加權計算得出。

## 六大評分維度與權重

| 維度 | 代號 | 權重 | 說明 |
|------|------|------|------|
| 供應商鎖定程度 | D1 | 25% | 現有供應商的綁定深度與排他性 |
| 採購決策複雜度 | D2 | 20% | 決策鏈層級、審批流程的複雜程度 |
| 技術門檻 | D3 | 20% | 對伺服器的客製化需求與技術認證要求 |
| 通路可及性 | D4 | 15% | ASUS 現有通路能否觸及該客戶 |
| 市場競爭強度 | D5 | 10% | 同時爭取該客戶的競爭對手數量與強度 |
| 地緣政治與合規風險 | D6 | 10% | 國安審查、資料主權、貿易限制等 |

## 各維度的具體評分標準

### D1：供應商鎖定程度（權重 25%）

| 分數 | 條件 | 典型情境 |
|------|------|----------|
| 1-2 | 無固定供應商，或正在積極尋找新供應商 | 新成立的 NeoCloud、剛獲得融資正在建置資料中心 |
| 3-4 | 有 2-3 家供應商但無排他合約，願意評估新方案 | 中型企業 IT，使用 Dell/HPE 但合約即將到期 |
| 5-6 | 有主要供應商但非獨家，切換成本中等 | 大型企業有全球框架合約，但各區域可獨立採購 |
| 7-8 | 深度綁定 1-2 家供應商，有長期合約或技術整合 | 與 Dell/HPE 簽署 3-5 年全球合約，含維護與支援 |
| 9-10 | 完全自研或獨家綁定，幾乎不可能切換 | Hyperscaler 自研伺服器（如 AWS Graviton）、國防機密系統 |

### D2：採購決策複雜度（權重 20%）

| 分數 | 條件 | 典型情境 |
|------|------|----------|
| 1-2 | CEO/CTO 可直接拍板，決策鏈極短 | 小型新創，創辦人即決策者 |
| 3-4 | 需 1-2 層審批，有明確的 IT 採購負責人 | 中型科技公司，CIO 可決定 |
| 5-6 | 需跨部門評估（IT + 採購 + 財務），3-6 個月週期 | 大型企業標準採購流程 |
| 7-8 | 需全球總部批准，涉及多國合規審查，6-12 個月 | 跨國金融機構、電信商的全球框架合約 |
| 9-10 | 需政府審批或董事會決議，超過 12 個月 | 國防承包商、國營企業、受制裁國家企業 |

### D3：技術門檻（權重 20%）

| 分數 | 條件 | 典型情境 |
|------|------|----------|
| 1-2 | 標準 x86 伺服器即可滿足，無特殊認證需求 | 一般企業 IT、Web Hosting |
| 3-4 | 需要特定 GPU 配置或高密度運算，但為市場通用規格 | AI 推論、HPC 標準配置 |
| 5-6 | 需要客製化韌體/BIOS、特定散熱方案或機櫃設計 | 大型雲端商的 OCP 規格、液冷需求 |
| 7-8 | 需要通過嚴格的安全認證（如 FedRAMP、ITAR）或自研晶片整合 | 政府雲、國防系統、自研 AI 晶片 |
| 9-10 | 完全自研架構，不接受外部標準伺服器 | Hyperscaler 自研（Google TPU、AWS Graviton）、Tesla Dojo |

### D4：通路可及性（權重 15%）

| 分數 | 條件 | 典型情境 |
|------|------|----------|
| 1-2 | ASUS 既有客戶或已有直接聯繫管道 | Nebius、Yotta、Sakura Internet 等既有客戶 |
| 3-4 | 可透過 ASUS 現有 SI/DIST 合作夥伴觸及 | 使用 CDW、WWT、SHI 等 ASUS 合作通路的企業 |
| 5-6 | 需要建立新的通路關係或透過展會接觸 | 需要透過 OCP Summit、KubeCon 等技術展會建立關係 |
| 7-8 | 封閉採購體系，僅接受特定合格供應商名單 | 大型電信商的合格供應商清單（AVL）、政府採購 |
| 9-10 | 完全封閉，無法透過任何現有管道接觸 | 軍事機密項目、受制裁實體 |

### D5：市場競爭強度（權重 10%）

| 分數 | 條件 | 典型情境 |
|------|------|----------|
| 1-2 | 競爭對手少，市場空白或 ASUS 有明顯差異化優勢 | 新興市場的 NeoCloud、ASUS 有獨家 GPU 配置 |
| 3-4 | 2-3 家競爭者，但 ASUS 在價格或交期上有優勢 | 中型企業市場，Dell/HPE 報價偏高 |
| 5-6 | 4-5 家競爭者積極爭取，需要差異化策略 | 標準企業 IT 市場 |
| 7-8 | 主要 OEM（Dell、HPE、Lenovo）全力爭取，價格戰激烈 | 大型企業的全球框架合約招標 |
| 9-10 | 客戶與競爭對手有策略聯盟或股權關係 | 如 Lenovo 與聯想集團、Dell 與 VMware 生態系 |

### D6：地緣政治與合規風險（權重 10%）

| 分數 | 條件 | 典型情境 |
|------|------|----------|
| 1-2 | 無地緣政治風險，自由貿易區域 | 北美、西歐、日本、澳洲的民營企業 |
| 3-4 | 輕微的資料主權要求，但不影響硬體採購 | GDPR 合規要求、資料在地化 |
| 5-6 | 需要特定國家的安全認證或在地化生產 | 印度的資料在地化要求、歐盟的 NIS2 指令 |
| 7-8 | 涉及出口管制或技術轉讓限制 | 中國的 GPU 出口管制、俄羅斯制裁 |
| 9-10 | 直接受到貿易制裁或國安審查 | 美國實體清單企業、國防敏感項目 |

## 計算公式

```
最終難易度 = round(D1 × 0.25 + D2 × 0.20 + D3 × 0.20 + D4 × 0.15 + D5 × 0.10 + D6 × 0.10)
```

最終分數四捨五入至整數，範圍 1-10。

## 評分範例

### 範例 1：AWS（NA #1）→ 難易度 10
- D1 = 10（完全自研伺服器，Graviton 晶片）
- D2 = 10（全球最大雲端商，決策極度複雜）
- D3 = 10（自研架構，不接受外部標準伺服器）
- D4 = 9（封閉採購體系）
- D5 = 9（所有 ODM 搶破頭）
- D6 = 2（美國企業，無地緣風險）
- 計算：10×0.25 + 10×0.20 + 10×0.20 + 9×0.15 + 9×0.10 + 2×0.10 = 2.5+2.0+2.0+1.35+0.9+0.2 = **9.0 → 9**

### 範例 2：Nebius Group（EMEA，ASUS 既有客戶）→ 難易度 3
- D1 = 2（ASUS 已是供應商）
- D2 = 3（中型企業，決策鏈短）
- D3 = 4（需要 GPU 伺服器但為通用規格）
- D4 = 1（既有客戶，直接聯繫）
- D5 = 4（有其他供應商競爭）
- D6 = 5（前 Yandex，有俄羅斯背景但已遷至荷蘭）
- 計算：2×0.25 + 3×0.20 + 4×0.20 + 1×0.15 + 4×0.10 + 5×0.10 = 0.5+0.6+0.8+0.15+0.4+0.5 = **3.0 → 3**
